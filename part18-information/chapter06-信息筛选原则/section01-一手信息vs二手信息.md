# 一手信息 vs 二手信息

## 定义

| 类型 | 定义 | 例子 |
|------|------|------|
| **一手信息** | 原始来源，未经过滤 | SEC文件、财报、电话会 |
| **二手信息** | 经过解读和加工 | 新闻报道、分析师文章、社交媒体 |

## 一手信息的优势

| 优势 | 说明 |
|------|------|
| 准确性 | 未经篡改 |
| 完整性 | 包含所有细节 |
| 时效性 | 最新发布 |
| 无偏见 | 未经过滤 |

## 常见一手信息

| 信息 | 来源 |
|------|------|
| 财务数据 | 10-K、10-Q |
| 管理层观点 | 财报电话会 |
| 风险披露 | 风险因素 |
| 机构持仓 | 13F文件 |
| 重大事件 | 8-K |

## 常见二手信息

| 信息 | 来源 |
|------|------|
| 新闻报道 | 财经媒体 |
| 分析师报告 | 投行研报 |
| 社交媒体 | Twitter、Reddit |
| 论坛讨论 | StockTwits |
| 博客文章 | 投资博客 |

## 二手信息的风险

| 风险 | 说明 |
|------|------|
| 偏见 | 作者可能有立场 |
| 不完整 | 可能遗漏关键信息 |
| 延迟 | 可能不是最新 |
| 错误 | 可能有事实错误 |

## 信息优先级

```
1. SEC文件（一手）
   └── 10-K、10-Q、8-K

2. 公司IR页面（一手）
   └── 财报、新闻稿

3. 财报电话会（一手）
   └── 管理层陈述+Q&A

4. 分析师报告（二手）
   └── 需要验证

5. 新闻报道（二手）
   └── 需要验证

6. 社交媒体（二手）
   └── 谨慎使用
```

## 实践建议

| 建议 | 说明 |
|------|------|
| 优先一手 | 先看原始文件 |
| 交叉验证 | 用多个来源验证 |
| 关注日期 | 确保信息最新 |
| 识别偏见 | 作者可能有立场 |
