# 收入确认

## 定义

> **收入确认（Revenue Recognition）**是指公司在何时、以何种金额将交易记录为收入。这是利润表中最关键的会计政策之一。

## 收入确认五步法（ASC 606）

| 步骤 | 内容 | 说明 |
|------|------|------|
| 1 | 识别合同 | 与客户之间的协议 |
| 2 | 识别履约义务 | 合同中的承诺 |
| 3 | 确定交易价格 | 预期收到的金额 |
| 4 | 分配交易价格 | 按比例分配到各履约义务 |
| 5 | 确认收入 | 履行义务时确认 |

## 收入确认时点

### 时点确认（Point in Time）

商品交付或服务完成时一次性确认。

| 例子 | 确认时点 |
|------|----------|
| 商品销售 | 交付给客户时 |
| 软件许可 | 授予许可时 |

### 时段确认（Over Time）

在一段时间内逐步确认。

| 例子 | 确认方式 |
|------|----------|
| SaaS订阅 | 按月/年确认 |
| 建筑合同 | 按完工百分比确认 |
| 服务合同 | 按服务提供进度确认 |

## 常见收入确认政策

| 行业 | 确认方式 | 关注点 |
|------|----------|--------|
| **SaaS** | 按订阅期分摊 | ARR、续费率 |
| **零售** | 交付时确认 | 退货率 |
| **建筑** | 完工百分比法 | 完工进度估计 |
| **硬件** | 交付时确认 | 渠道库存 |
| **广告** | 展示/点击时确认 | 效果计量 |

## 收入确认的陷阱

### 1. 提前确认收入

在商品未交付或服务未完成时确认收入。

### 2. 渠道填塞（Channel Stuffing）

向经销商大量发货，确认收入，但经销商并未卖给终端消费者。

### 3. 循环交易

公司A卖给公司B，公司B再卖回给公司A，双方都确认收入。

### 4. 附条件销售

销售附有退货权、回购条款等，但仍全额确认收入。

## 如何识别收入确认问题

| 信号 | 可能含义 |
|------|----------|
| 应收账款增速 > 收入增速 | 可能在提前确认收入 |
| 季末收入暴增 | 可能在渠道填塞 |
| 收入增速远超同行 | 需要验证真实性 |
| 频繁更改收入确认政策 | 可能在操纵利润 |
| 退货率上升 | 之前确认的收入可能要冲回 |

## 典型案例

### 问题案例：Lucent Technologies

- 向渠道大量发货，确认收入
- 渠道库存积压，后续大幅退货
- 收入和利润虚增

### 正面案例：SaaS公司

- 按订阅期均匀确认收入
- 收入可预测、可重复
- 高质量的收入模式

2. 如何判断一家公司的收入确认是否激进？
3. 渠道填塞为什么不可持续？
